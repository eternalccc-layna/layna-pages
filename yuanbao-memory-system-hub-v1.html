四类记忆

工作 · 情景 · 语义 · 程序。每个主流框架都基于这套分类。给 GPT-3.5 加一层认知架构，编码基准从 48% 拉到 95%。

MemGPT · UC Berkeley 2023

LLM 即操作系统

把上下文窗口当 RAM、外部存储当磁盘。三层结构：核心记忆（2KB 常驻）/ 回忆记忆（搜索）/ 归档记忆（无限）。

三层模型 · 业内通行

瞬时 / 长期 / 技能

瞬时层管「当下流畅」，长期层管「沉淀理解」，技能层管「能拿来用」。8 种细分记忆都能塞回这三层。

03 — 行业难题

分层之后，
真正的难点才开始。

业内文章反复点出三个共同病灶——这也是评估任何记忆方案优劣的标尺。

01

覆盖式遗忘

No Versioning

新信息直接覆盖旧信息，历史的因果链条被抹掉。

结果：Agent "改主意"了你看不到；半年前用户说过的偏好突然失效但不知道为什么。

02

堆积式噪声

No Consolidation

所有信息一股脑堆进库里，没有合并、没有蒸馏。

结果：长期使用后偏好混乱、矛盾事实并存、检索结果参差不齐。

03

相似度错位

Recency vs Relevance

只靠向量相似度做检索，但相似 ≠ 相关。

结果：把三个月前的旧场景当成今天的关键证据，关键的近期决策反而召回不到。

04 — 关键机制

好的记忆系统，
都同时做对了这四件事。

分层只是地基。真正能跑起来的记忆，靠的是层与层之间的转换 / 检索 / 遗忘 / 巩固。

① 巩固 Consolidation

从情景到语义

把"那天发生了什么"蒸馏成"这件事的规律"。
客服 Agent 处理上百次退货 → 抽出语义规则「7 天内损坏可极速换货」。

这一步通常靠开发者写规则或周期性 LLM 摘要。脆弱、难验证、最被忽视，但决定记忆系统的天花板。
② 检索 Retrieval

相关性 + 时效 + 重要性

单看相似度不够，必须三维加权。
  • 相关性　向量相似度，主导默认排序
  • 时效　近期记忆给更高权重，避免「捡到旧的」
  • 重要性　高奖励 / 高惊讶事件标记为高权值
③ 遗忘 Forgetting

主动剪枝，不是被动溢出

遗忘是健康记忆的一部分，不是漏洞。
  • 摘要压缩　长尾对话归档前先压成简短摘要
  • 重要性剪枝　低于阈值的记忆按周期淘汰
  • 聚类合并　语义近似的记忆合并降冗余
④ 控制 Control Policy

谁来决定存什么

业内最有分歧的一题，也是最关键的一题。
程序式　开发者写规则决定。可控，但用户场景一变就失效。

Agent 自管　Agent 把store / retrieve / update / discard当作可调用工具。 MemGPT、Letta 都走这条路——成本高，但能学到「上下文写满前先摘要」这类非显然策略。
05 — 落到我们

那么，
「AI 资产中枢」要做什么？

把上面 4 类记忆 × 4 个机制，对齐到我们「多产品矩阵 + 5 类资产」的现实底座上——这才是 Asset Hub 真正的产品形态。

Asset Hub · AI 资产中枢

不是收藏夹的升级，
是矩阵的「记忆层」。

把 5 类资产（链接 / 笔记 / 文件 / 对话 / 任务）按记忆系统的分层重新组织，让每一个产品都能从同一份记忆里读取上下文，又能反过来贡献新事实。

引擎层接 Hy-Memory，
产品层是我们的战场。

记忆系统已经有了成熟的认知架构和工程方案——四类记忆、三层栈、四个机制。
我们要做的，不是再发明一套，而是把它翻译成多产品矩阵的产品形态：
让用户看得见自己的偏好，让 5 类资产同时是情景与语义，让每一个端都从同一份记忆里读取上下文。
