02 · Open Questions

下午要拍板的 3 个核心问题

每个问题都有"我方倾向"作为开场白，避免会议跑偏。

QUESTION 01 · 元宝侧改造意愿

「绑定→解绑后视为已注册，UID 不可回绑」是否有计划改造？

背景
元宝当前逻辑：用户解绑微信后，wx_openid 仍标记为"已注册"，新用户用同一微信再注册会失败，且不能回绑到原 UID。
影响
用户解绑微信再回绑 → 直接登录失败 / 创建新 UID → 同一自然人产生多 UID（对应风险表场景 7 🔴）。
我方倾向
推动元宝排期改造：解绑后释放凭证，允许同一 openid 回绑同一 UID。若元宝短期不能改，新矩阵走方案 3，用 BrowserID 自有体系解耦。
要拿结论
元宝是否有改造排期？如有，预计什么时候？如果没有，是否能给到一个解绑兜底接口（如允许 admin 后台手动回绑）？
QUESTION 02 · 方案 2 的研发支持

若选方案 2，元宝能否支持"按新矩阵规则分配 UID"？

背景
方案 2 需要元宝侧暴露 UID 生成接口，并允许新矩阵在注册阶段调用。
影响
如果元宝 UID 生成不可定制，新矩阵的"游客模式 → 登录态升级"流程会很别扭。
我方倾向
不优先选方案 2。改造涉及元宝核心账号链路，风险高、排期长。倾向直接走方案 3 解耦。
要拿结论
如果元宝坚持方案 2，请给出具体的接口形态、SLA、改造工期；若超过 2 个月，新矩阵直接走方案 3。
QUESTION 03 · 多账号绑定状态同步

若用户在元宝内已绑多账号，新矩阵登录后是否完全同步？

背景
用户在元宝内可能同时绑了微信 + QQ + 手机号；解绑、换绑、注销时新矩阵侧如何同步？
影响
不同步 → 用户在元宝解绑微信，新矩阵仍允许微信登录到旧 UID，体感割裂；过度同步 → 元宝侧任何变更直接影响新矩阵，耦合过深。
我方倾向
分级同步：① 注销 = 强同步（必须立即冻结新矩阵资产）② 解绑 / 换绑 = 弱同步（异步通知 + 新矩阵自决策）③ 新增绑定 = 不同步（用户在新矩阵主动触发）。
要拿结论
元宝能否提供"账号事件订阅"机制（webhook / MQ）？事件粒度（绑/解绑/换绑/注销 4 类）能否承诺？

05 · Compliance Assessment

合规评估 · 同体系不同 App 间能否完全互通？

Q1 · 法规依据
《个人信息保护法》第 23 条 · 单独同意
向其他个人信息处理者提供个人信息的，应当向用户告知接收方名称、联系方式、处理目的、处理方式和个人信息种类，并取得单独同意。
Q2 · 监管口径
网信办《App 违法违规收集使用个人信息》
同一公司不同 App 之间共享账号信息，仍属"信息共享"行为。需在隐私政策第 X 章节明确披露，不能默认全通。
Q3 · 默认互通是否合规
不合规 ❌
即便都是腾讯系，元宝 / 浏览器 / 输入法之间共享账号必须用户授权。建议登录后弹出"是否将账号资产同步到 X App"的二次确认。
Q4 · 推荐做法
分级授权 + 撤销通道
① 默认仅同步"登录态"（最小集）
② AI 资产 / 收藏等需用户开关确认
③ 设置中提供"已授权 App 列表"+ 一键撤销。
Q5 · 注销联动
用户在 A 注销 ≠ B 自动注销
合规要求：元宝注销 → 新矩阵询问用户是否一并注销 / 仅断开关联 / 保留独立账号。三选一不能默认。
Q6 · 跨境合规
出海版本需独立账号体系
GDPR / CCPA 对账号互通要求更严。出海版（P2）建议拆出独立 BrowserID 体系，不与国内打通。
06 · Action Items

下午会议要从元宝拿到的 7 件事

按优先级排序 —— 拿不到红色 3 项，会议视为未达成对齐。

  1. 🔴 P0  注销回调机制：元宝 UID DELETING / DELETED 时，能否给新矩阵发 webhook？SLA 多少？
  2. 🔴 P0  多 UID 合并能力：同一手机号产生两个 UID 时，元宝是否提供合并 API？合并不可逆还是可回滚？
  3. 🔴 P0  解绑 = 注册的改造意愿：是否有计划放开"同一 openid 回绑同一 UID"？如有排期请同步；如无，新矩阵走方案 3 解耦。
  4. 🟡 P1  UID + status 接口：能否在 UID 查询接口里直接返回 status 字段（ACTIVE/DELETING/DELETED）？
  5. 🟡 P1  账号事件订阅：绑定 / 解绑 / 换绑 / 注销 4 类事件能否独立订阅？粒度与延迟？
  6. 🟡 P1  双向凭证表查询：能否提供"UID → 已绑凭证列表"接口？用于注册凭证丢失时的兜底找回。
  7. 🟢 P2  方案 2 接口形态评估：若坚持方案 2，请给出 UID 生成接口形态和工期 —— 超过 2 个月则确定走方案 3。
