04 三个故事的共同脊柱

不同人群、不同场景，同一套底座规律。剥开故事壳，能看到三件确定的事。

脊柱 故事① 追热点 故事② 会议复盘 故事③ 出行筹备
起点 = 看到 浏览器看到论文 录音笔录到对话 浏览器刷到攻略
中段 = 加工 输入法 + 翻译 协同 翻译 + 浏览器 整合 文件管家 + 日历 编排
终点 = 沉淀 浏览器笔记（选题文档） 浏览器纪要 + 日历任务 浏览器游记 + 朋友圈分享
穿线 = ID YuanbaoID 透传 YuanbaoID 透传 YuanbaoID 透传
枢纽 = Hub 链接 → 笔记 → 引用 对话 → 纪要 → 任务 链接 + 文件 → 任务 → 笔记

🎯 三句话总结

① 账号是地基：用户感知不到 ID，但没有 ID 就没有"接得上"。

② 资产是货币：5 类资产在 6 端之间流动，每流动一次价值翻一倍。

③ 浏览器是枢纽：所有故事的起点或终点都在浏览器 — 这是元宝矩阵的"主战场"。

对外一句话：「在元宝矩阵里，你不是在用 6 个 App，你是在用 1 个底座的 6 个表达。」
